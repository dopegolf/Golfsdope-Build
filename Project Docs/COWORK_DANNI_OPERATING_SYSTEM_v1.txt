## PART 8: COMMON TASK PLAYBOOKS

**Playbook A: Build a new page section on Hostinger**
1. Review the approved copy for that section
2. Open the Hostinger builder tab
3. Navigate to the correct page
4. Click "Add Section" or equivalent button in the builder
5. Choose the right layout (hero, text+image, email capture, etc.)
6. Drag in the image from the desktop folder (or Google Drive if using external assets)
7. Fill in the text, match the color/brand system
8. Preview on mobile to check it looks right
9. Take a screenshot, send to Ronnie with "Section [name] ready for review" message
10. Wait for approval before moving to the next section

**Playbook B: Fix a social link**
1. Find the social icon/link on the live page
2. Click on it to edit
3. Replace the placeholder URL with the real account URL
   - Instagram: https://instagram.com/dopegolfcompany
   - Facebook: https://facebook.com/dopegolfcompany (or full company page URL once confirmed)
   - YouTube: https://youtube.com/@dopegolfcompany
4. Preview to confirm it opens the right account
5. Report: "Social links updated — checked all four, all now point to real accounts"

**Playbook C: Swap an image**
1. Identify which image needs to be swapped (resolution, brand fit, etc.)
2. Find the right replacement in the desktop folder
3. In Hostinger builder, click on the image to edit
4. Upload/replace with the new image from the folder
5. Adjust sizing/positioning if needed (maintain aspect ratio, don't distort)
6. Preview on mobile
7. Report: "Image swapped from [old] to [new], looks sharp on mobile"

**Playbook D: Review a page against brand system**
1. Open the live page in Hostinger preview (or live if already published)
2. Take a screenshot
3. Check against these questions:
   - Colors match (black, gold, ember/copper accents)?
   - Logo/crest consistent?
   - Real social accounts linked (not placeholders)?
   - Taglines accurate ("Play Dope. Stay Dope." etc.)?
   - No fake promises (product claims, dates, inventory)?
   - Photography style consistent (lifestyle > product-on-white, except shop pages)?
   - Mobile looks good?
4. Document any mismatches
5. Report to Ronnie with findings and a screenshot
